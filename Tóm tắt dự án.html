Bối cảnh & Tính cấp thiết

  • Khủng hoảng toàn cầu: Đột quỵ là nguyên nhân hàng đầu gây tử vong và tàn tật trên thế giới.
  • Thống kê báo động: Khoảng 15 triệu ca mắc mới mỗi năm, dẫn đến 5 triệu ca tử vong.
  • "Giờ vàng" (Golden Hour): Can thiệp trong 3-4.5 giờ đầu giúp giảm đáng kể tỷ lệ tử vong và biến chứng.
  • Giải pháp đề xuất: Ứng dụng AI và công nghệ di động để phát hiện sớm và phản ứng nhanh.

Kiến trúc Hệ thống

Kiến trúc Phân lớp (Layered Architecture)

  • Tầng Giao diện (Presentation): Flutter Mobile App (Android/Web).
  • Tầng Nghiệp vụ (Business): 34 lớp Service xử lý logic cốt lõi.
  • Tầng Dữ liệu (Data): Firebase (Realtime DB, Auth) & Hive (Cache cục bộ).
  • Tầng AI: Python Flask Server chứa mô hình Random Forest.

Công nghệ Sử dụng

Frontend & Di động

  • Flutter SDK 3.5.4+: Phát triển đa nền tảng.
  • Dart: Ngôn ngữ lập trình cốt lõi.
  • fl_chart: Trực quan hóa dữ liệu.
  • Hive: Lưu trữ dữ liệu offline.

Backend & AI

  • Firebase: Auth, Realtime DB, Cloud Messaging.
  • Python (Flask): Dịch vụ API cho AI.
  • Scikit-learn: Thuật toán Random Forest Classifier.
  • Google Maps API: Dịch vụ vị trí cho SOS.

Triển khai Module AI

Mô hình: Random Forest Classifier

Phương pháp học máy tổ hợp (ensemble learning) phù hợp cho phân loại y tế nhờ độ chính xác cao và khả năng chống overfitting.

  • Đầu vào: 14 đặc trưng sức khỏe (Tuổi, BMI, Glucose, Huyết áp, Bệnh tim, Loại công việc...).
  • Đầu ra: Xác suất rủi ro (%) và Mức độ cảnh báo (Thấp, Trung bình, Cao).
  • Triển khai: Host qua Flask API để suy luận thời gian thực.

API AI - Kiến trúc Backend

Flask REST API Server

  • Server: Flask chạy trên port 5001 với Debug mode.
  • Model AI: AdaBoostClassifier với ColumnTransformer preprocessor.
  • Endpoints:
    • /health - Kiểm tra trạng thái server
    • /predict - Dự đoán nguy cơ đột quỵ
  • Tích hợp: Chạy song song với Flutter Web App.
  • Network: Hỗ trợ localhost và LAN (192.168.x.x).
